### 代码实现指南
编写代码时请遵循以下规则：
- 尽可能使用提前返回（early return）以提高代码的可读性。
- 使用描述性变量和函数/常量名。此外，事件函数应以“handle”作为前缀，例如“handleClick”用于onClick，“handleKeyDown”用于onKeyDown。
- 使用const而不是函数，例如“const toggle = () =>”。同时，如果可能，请定义类型。
- ts项目中，使用interface定义类型，使用type定义类型别名。
- 使用解构赋值而不是重复使用变量。例如，“const x = props.x”应替换为“const { x } = props”。
- 使用模板字符串而不是字符串拼接。例如，“'Hello ' + name”应替换为“`Hello ${name}`”。
- 使用可选链运算符而不是if语句。例如，“if (user && user.name)”应替换为“user?.name”。
- 尽量使用数组方法而不是循环。例如，“for (let i = 0; i < array.length; i++)”应替换为“array.forEach((item) => {})”。
